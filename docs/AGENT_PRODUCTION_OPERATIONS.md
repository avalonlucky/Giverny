# Agent 生产控制台

> 本手册、根目录《使用手册》和 `CHANGELOG.md` 会在每次构建时自动生成 Agent 产品知识索引。管理员更新并发布这些文档后，工作助手即可检索相应的产品流程、运营方法和版本事实。

本文用于向管理员和使用者讲解 Giverny Agent 的持续执行、记忆、提醒、失败学习与质量监控。它描述的是产品行为，不是模型宣传。

## 1. 全生命周期自动验收

发布门禁会在临时 D1 中真实执行：创建任务、记录进展与工时、记录等待、编辑 / 删除单条记录、标记验收文件、填写验收备注并完成验收。所有测试数据都在隔离目录中，带评测标记的请求不会进入正式运行指标。

它验证的是数据一致性和安全边界，不只是判断回答里有没有关键词。任何一步写入失败、重复执行产生重复数据、验收状态与进度不一致，都会阻止发布。

## 2. 持续计划

用户可以说“为任务 #123 制定从当前状态到验收的计划”。Agent 把目标保存为多步计划；后续确认执行相应工具后，匹配步骤会自动完成。

任务中心允许：

- 人工完成或重新打开某一步。
- 暂停计划，停止自动推进，但不修改任务。
- 恢复已暂停或已完成计划。
- 取消不再需要的计划。

持续计划不是无人值守写入。涉及任务、进展、反馈和验收的每次业务修改仍必须生成确认卡并由用户确认。

## 3. 可控长期记忆

任务记忆按任务隔离，自动压缩需求、近期进展、甲方反馈和未解决事项。任务详情仍是权威事实，记忆只用于减少重复解释。

用户可以：

- 增加人工纠正，例如“甲方偏好简洁排版”。
- 忽略已经解决或判断错误的待办。
- 恢复此前忽略的待办，让系统重新计算。
- 清空并停止某个任务的记忆；重新启用后从权威任务数据重新生成。

记忆不会跨任务混用，也不能覆盖任务原始字段、工时或验收记录。

## 4. 主动提醒与快捷处理

每天检查逾期、进度 100% 但未验收、实际工时超过预估 25%、存在等待记录、已有验收文件但缺验收备注等信号。提醒进入任务中心并与任务 ID 关联。

点击“执行建议”后，系统会按提醒类型生成明确指令，例如检查逾期原因并生成进展草稿、核对资料并生成验收草稿。Agent 必须先读取真实任务数据，再返回可确认操作。

## 5. 失败学习与回归

系统只保存匿名失败指纹：失败类别、意图类型、首个工具、HTTP 状态、次数与时间。同一指纹出现两次后进入“待回归”。管理员可以标记为已覆盖、重新加入回归或忽略。

失败学习不保存问题原文、模型回答、任务名称、附件内容和确认草稿。它的目的不是让模型自行修改代码，而是把高频失败变成下一批自动化测试输入。

## 6. 质量、Token 与参考成本

设置页按 7 天或 30 天展示成功率、工具调用率、P95 / 平均响应、审批与消歧次数、模型回落和失败次数。Token 根据请求与结构化响应字符量近似计算；参考成本使用内置模型单价估算。

参考成本只用于比较趋势和模型路线，不替代 DeepSeek、豆包、Gemini 等供应商账单。涉及财务核算时必须以供应商控制台为准。

## 7. 七天后再调模型

模型建议至少需要 7 天且 30 次真实请求。达到门槛后，系统根据失败率、模型回落率、P95 响应和任务消歧率给出建议。样本不足时只显示观察进度，不建议切换模型。

建议不会自动修改模型、API Key 或路由。管理员应先处理待回归失败，再比较不同模型的成功率、平均延迟、Token 和参考成本，最后人工决定是否调整。

## 安全原则

- 所有业务写入继续执行“预览 → 确认 → Workflow”。
- 任务记忆可查看、可纠正、可清空、可停用。
- 失败学习与质量指标不保存业务正文。
- 模型调优建议只读，不自动改配置。
- 正式任务数据、附件和结算永远以 D1 / R2 权威记录为准。
