# UI 排查记录 · 2026-06-26

本轮按颜色、按钮、组件、布局四个方向排查，并优先处理「洞察」总结报告的表达价值问题。

## 当前结论

### 颜色

- 主页面 UI 基本已使用 `App.css` 顶部的 `--color-*` token，任务状态、验收、补录、管理员专属信息等语义色有明确来源。
- 仍有少量裸色值集中在导出、Canvas、图表调色板和历史兼容样式里；这些不一定马上影响页面观感，但后续应逐步迁移成 chart / export 专用 token。

### 按钮

- 全站已有 `primary-button`、`ghost-button`、`text-button`、`icon-button` 等基础按钮语言，关键动作基本能复用。
- 仍需继续收敛右键菜单、文件操作和局部工具按钮：这类地方容易出现“看起来像临时补丁”的重按钮，应优先改成轻文字或图标操作。

### 组件

- Modal、日期选择、进展记录、附件上传、进度条等高频组件已基本复用。
- 洞察、结算、收入等分析型页面还存在一些局部专用结构；后续新增内容应继续走“文档式排版”，少用卡片、阴影和装饰性边框。

### 布局

- 主工作台和右侧任务面板已逐步回到方正、留白、横线分隔的体系。
- 洞察、收入、结算仍是最容易显得繁琐的区域：统计信息要减少“块状堆叠”，尽量用横向数据行、文本结论和清晰表格承载。

## 本轮已落地

- 「洞察 → 总结报告」新增周报内容：完成事项、投入情况、下步计划。
- 新增年度盘点式指标：最晚奋斗时间、最吃工时任务、深夜仍在线。
- 新增「夜间小传记」：把 22:00 后或 06:00 前的新建任务、验收进展、附件上传提炼出来，帮助复盘真实工作节奏。
- 新增内容采用横线和文字层级，不新增重卡片。

## 后续优先级

1. 继续把收入和结算里的统计区拍平，弱化卡片感。
2. 将图表 / 导出里的裸色值迁移为语义 token。
3. 清理右键菜单和文件操作中的重按钮，保留必要动作即可。
4. 对窄列长标题、长文件名、长备注做一次统一省略和展开策略检查。
