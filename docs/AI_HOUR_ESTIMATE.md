# AI 工时建议

更新时间：2026-07-16

## 目标

AI 工时建议用于在新建或编辑任务时，结合当前任务范围和历史真实投入，给出可解释、可校准、可持续学习的排期参考。

它不代替分段计时，也不直接参与结算。结算仍只读取任务保存后的实际工时。

## 输入

每次分析会读取：

- 任务名称、设计类型和具体需求。
- 需求人。
- 预计开始、预计交付和当前手工预估。
- 甲方需求附件名称与可提取文字。
- 已验收历史任务的需求、实际工时、进展、验收备注和反馈。
- 历史 AI 建议、最终采用值和验收后的真实工时。
- 用户对建议给出的“偏低 / 合适 / 偏高”评价及原因标签。
- 用户对历史参考任务给出的“并不相似”反馈。
- 站点当前小时单价，用于把预测工时转换为报价参考。

## 1. 复杂度画像

Worker 使用确定性规则先提取统一画像，模型不能反向改写这些证据：

| 维度 | 主要信号 |
|---|---|
| 设计基础 | 从零设计、全新视觉、复用旧稿、延续系列、套版、改字换图 |
| 交付规模 | 张数、页数、版本数、套数、交付单元数量 |
| 内容准备 | 文案素材齐全、附件已提供、待补、未定、需整理或提炼 |
| 版本适配 | 横竖版、多尺寸、多平台、多端、多比例 |
| 专项处理 | 插画、信息可视化、精修合成、三维、动效视频、复杂排版 |
| 协作风险 | 加急、多人确认、方向未定、多轮改稿 |

各维度汇总为 0–100 分：

- 低复杂度：小于 42。
- 中复杂度：42–69。
- 高复杂度：70 及以上。

复杂度分数用于温和调整历史统计基线，调整范围受限，避免单条关键词造成倍数级波动。

## 2. 分层检索与重排

候选样本只取“已验收且实际工时大于 0”的有效任务。

检索顺序：

1. 精确同设计类型。
2. Vectorize 需求语义相似任务。
3. 前两层不足 3 条时，补少量同大类任务。

候选召回后再按以下因素重排：

- 类型关系。
- 向量语义分数。
- 复杂度画像相似度。
- 交付数量、页数和适配数量差异。
- 从零 / 复用是否一致。
- 专项处理重合度。
- 内容准备程度。
- 是否同一需求人。
- 样本时间：90 天内的近期任务保持完整权重，较早任务逐级衰减；缺少实际交付时间时依次回退到预计交付和开始时间。

最终统计使用重排后的权重计算加权平均、P25、中位数和 P80。同大类样本始终保持弱权重，不能与精确同类混为同等证据。

用户可以在参考任务后标记“不相似”。该反馈只对同设计类型、复杂度仍处于相近区间的后续分析生效，把该样本权重降为原来的 25%，但不删除历史任务，也不允许一次点击全局屏蔽样本。

## 3. 数值建议

常规建议以加权中位数为主要基线，并结合：

- 当前任务复杂度温和系数。
- 同类型历史“真实工时 / AI 建议”偏差。
- 达到门槛后的需求人校准系数。
- 模型对当前需求和历史备注的有限解释微调。

模型输出受到历史 P25、P80、最大值和当前手工预估构成的边界约束。模型置信度只能降低数据置信度，不能把低质量样本抬成高置信度。

稳妥预留取以下值中的较高者：

- 历史 P80。
- 常规建议加置信度风险余量。
- 常规建议至少增加 0.5 小时。

页面同时展示常规预测区间。下界参考历史 P25，上界参考 P80、稳妥值和本次校准系数。区间不是承诺值，而是用于表达历史波动和当前需求不确定性。

## 4. 可解释拆分

常规建议会按任务画像拆为：

- 核心设计。
- 批量制作 / 页面延展。
- 内容整理。
- 版本适配。
- 专项处理。
- 沟通与改稿。
- 交付整理。

拆分总和与常规建议一致。它用于解释建议来源，不会创建实际工时记录。

## 5. 需求人校准

需求人校准只使用该需求人已验收且同时存在预估和实际工时的任务。

最低门槛为 3 条完整样本。达到门槛后计算：

- 历史实际 / 预估比率的中位数。
- 平均改稿轮次。
- 需求完整率。

系数限制在 `0.8–1.3`，并与同类型全局校准共同受最终预测边界约束。未达到门槛时只展示观察，不改变建议值。

## 6. 个人采用校准

任务保存时，系统自动记录“AI 常规建议值”和“用户最终采用值”。同一设计类型达到 3 次可比较记录后，会提取稳定的上调或下调偏好：

- 直接采用建议，记为一致。
- 修改后保存，使用最终采用值 / AI 建议值作为纠偏样本。
- 面板中的“偏低 / 合适 / 偏高”作为任务尚未验收前的轻量反馈。
- 多次反馈属于同一次 suggestion 时按最新记录去重。
- 较早采用记录会温和衰减，系数限制在 `0.85–1.2`。

个人采用校准不能覆盖已验收真实工时，只能在历史统计基线上做受控调整。

## 7. 历史命中率与风险

当同类型建议已经回写验收实际工时后，页面会展示：

- 最近可用预测中误差不超过 20% 的比例。
- 绝对误差率的中位数。
- 本次不确定因素，例如设计基础未知、交付数量未知、素材准备未知、改稿风险、加急、样本不足或历史分布较散。

命中率只使用 `status = observed` 且建议值、实际值均有效的记录，不使用未验收任务伪造准确率。

## 8. 缺失信息提醒

当需求未明确以下内容时，接口会返回最多 4 个澄清问题：

- 从零还是复用。
- 交付数量或页数。
- 文案素材准备程度。
- 多尺寸 / 多平台适配。
- 确认方和修改轮次。

用户补充任务需求或附件后，应重新运行 AI 分析；旧建议会被标记为过期，不能直接采用。

## 9. 持续学习

`hour_estimate_suggestions` 保存：

- 输入指纹和任务上下文。
- 常规值、稳妥值、置信度。
- 复杂度画像和工时拆分。
- 参考任务与匹配原因。
- 用户最终采用的预估工时。
- 建议质量评价和偏差原因。
- 验收后的实际工时。

完整生命周期：

```text
需求输入
  -> 画像与检索
    -> 建议与解释
      -> 用户采用或修改
        -> 轻量准确性反馈
        -> 真实分段计时
          -> 验收结果回写
            -> 同类型与需求人校准
```

验收发生时，Worker 会自动生成一条结果复盘，记录常规值、稳妥值、最终采用值、真实工时、误差方向和可验证偏差因素。偏差因素来自任务事实，包括改稿轮次、等待时长、素材缺失、多版本适配、专项处理、加急和反馈标签，不由模型自由猜测。

## 10. 准确率与校准仪表盘

管理员可在「洞察 → AI 工时学习 → 准确率与校准」查看当前月份的学习结果：

- 已复盘预测数、误差不超过 20% 的命中率、中位误差和平均误差。
- 采用常规值、稳妥值或手工修改后，各自的使用次数和中位误差。
- 按设计类型、需求方 / 客户汇总的样本量、命中率、校准系数、平均改稿轮次和需求完整率。
- 最近已验收任务的逐任务复盘，包括建议值、采用值、真实工时、偏高 / 偏低方向和偏差因素。

仪表盘只读取已验收且已回写真实工时的建议。类型或需求方达到 3 条样本后才建议应用独立系数；未达到门槛时保留观察数据，不参与自动调整。

“采用后改善”比较最终采用值和 AI 原始建议相对真实工时的中位误差差异，用来判断稳妥值或人工修正是否真的让估算更准确，而不是只统计按钮点击次数。

## 11. 学习 2.0：趋势、人工校正与版本对照

仪表盘会同时展示最近 12 个月的命中率、中位误差、低估率和高估率。单月数据用于定位当前问题，跨月趋势用于判断系统是否真的持续变准。

每条近期复盘都可以由管理员人工校正偏差原因，例如需求中途追加、改稿轮次增加、素材延迟、多版本或专项制作。人工校正会覆盖自动归因，并作为后续同类任务的复盘上下文；它不会改写真实工时。

系统会比较 AI 分析时的原始需求和任务验收时的最终需求。若交付数量、页数、适配版本、专项处理或需求文本发生显著扩张，会记录需求变化摘要，并从同类型偏差系数和需求人校准中排除，避免把范围追加误判为算法低估。

每次建议都会保存算法、提示词和模型路由版本。洞察中的“预测版本对照”按真实验收结果比较各版本的样本数、20% 命中率和中位误差，新版本不能只凭主观体验宣称更准确。

## 12. 报价建议

工时分析使用站点当前小时单价同步计算：

- 常规报价：常规建议工时乘以小时单价。
- 稳妥报价：稳妥工时乘以小时单价。
- 报价区间：常规预测区间的上下界分别乘以小时单价。
- 风险预留：按置信度和本次风险因素数量给出 3%–25% 的解释性参考。

报价只用于决策参考，不会自动写入任务、修改小时单价或生成结算记录。

## 安全边界

- 不使用未验收任务作为权威结果。
- 不把自然交付周期换算成实际工作量。
- 不自动修改任务、实际工时或结算金额。
- 不允许模型自行更新业务规则、数据库结构或提示词。
- 样本不足时降低置信度，不用全站无关任务硬凑。
- `basis_json` 保存分析快照，便于后续审计当时为什么给出该建议。
- 同一次建议的多条反馈按 suggestion id 去重，避免重复点击放大权重。
- 显著需求扩张样本不参与预测偏差和需求人效率校准。
- 人工原因校正、样本相似度反馈和版本快照均保留审计依据，但不会覆盖业务事实。

## 用户验证建议

使用同一设计类型准备三组任务进行对照：

1. 单张复用旧版、素材齐全。
2. 单张从零设计、素材齐全。
3. 从零设计、3 个尺寸适配、文案待整理。

预期结果：

- 第二组复杂度和建议值不低于第一组。
- 第三组出现版本适配、内容整理和改稿风险相关拆分。
- 参考任务说明能解释类型、语义和复杂度为何相似。
- 同一需求人完整样本不足 3 条时不应用独立系数。
- 同一类型采用记录不足 3 条时不应用个人采用系数。
- 较早样本的匹配原因显示已降权，近期任务显示“近期已验收任务”。
- 常规区间上下界有效，历史命中率只在存在验收回写后展示真实统计。
- 任务验收后，建议记录能获得最终实际工时并参与后续校准。
- 「洞察 → AI 工时学习」能看到该任务的自动复盘，且类型 / 需求方样本达到 3 条后才展示可应用校准系数。
- 把参考任务标记为“不相似”后重新分析，同复杂度区间内该样本应显示“用户曾校正为不相似”并明显降权。
- 扩大任务需求后再验收，复盘应显示需求变化，且该样本不污染类型和需求人校准。
- 报价金额应严格等于工时乘以当前小时单价，且不会自动写入结算。
