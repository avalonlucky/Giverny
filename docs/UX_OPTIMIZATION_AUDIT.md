# Giverny 产品交互优化走查

更新时间：2026-06-15 22:41

## 优化原则

- 高频主操作保留清晰按钮，例如新建任务、锁定结算、确认验收。
- 熟悉度高的工具操作优先图标化，例如关闭、预览、打开、复制、删除、筛选。
- 次级操作优先放进右键菜单、二级菜单、折叠区或浮层，减少常驻占位。
- 工作台、任务页和文件库中相同语义的交互优先共用组件，避免一个地方改了另一个地方漏掉。
- 所有图标按钮必须提供 `aria-label` 和 `title`，让鼠标悬停和辅助工具能读懂含义。

## 已完成

- 弹窗右上角关闭按钮统一为圆形图标按钮，去掉可见的“关闭”文字。
- 新建任务的“补录”入口移到弹窗右上角小开关，打开后才显示结算月份浮层。
- 补录月份确认后收起，并在开关旁显示当前记录月份，避免遮挡表单。
- 新建任务的预计开始 / 预计交付支持手动输入日期，同时保留日历按钮。
- 工作台任务明细移除重复筛选图标菜单，只保留状态分段 Tab 作为唯一筛选入口。
- 顶栏移除重复的“月度结算单”入口，侧边“月报”改名为“结算”。
- 结算页的结算历史默认只展示最近一条，多条历史可展开 / 收起。
- 工作台任务列表和任务页列表共用同一个任务右键菜单组件。
- 任务右键菜单增加“状态”二级菜单，把“进行中 / 待验收”收进二级层级。
- 工作台右侧栏的工作分段与等待记录均提供「编辑 / 删除」操作；删除前使用站内确认框，工作分段删除后同步重算实际工时，等待记录删除后不再进入洞察分析。
- 文件预览弹窗去掉底部重复关闭按钮。
- 文件库文件卡片加入右键菜单，支持预览、打开、重命名、添加标签和删除。
- 左侧导航曾尝试加入高频 / 低频分隔线和「结算」「收入」悬停说明，复核后已移除；当前 6 项导航保持连续，更符合自用工具的轻量感。
- 工作台和任务页空状态加入「新建任务」入口。
- 任务详情移除重复快捷状态按钮，保留状态下拉和「确认验收」主按钮。
- 收入页税务估算参数默认折叠，设置页将业务设置前置、Cloudflare 绑定默认折叠；「结算设置」已改为整行铺满、四项等宽的紧凑布局。
- 「默认客户」改为「服务公司名称」，并作为可编辑业务设置同步到结算回单和分享查看页。
- 结算回单新增「工时明细附表」，在保留摘要表的同时补齐 Excel 工时表字段，打印时使用 A4 横向版式承载 16 列明细。
- 设置页「设计类型」大类改为桌面双列排布，并支持大类名称直接编辑；相比三列，双列能节省滚动又不压缩子类输入区。
- 左侧底部移除 R2 文件空间卡片和「试运营」字样，只保留小版本号；R2 与 Cloudflare 信息统一收纳到设置页「系统资源」。
- 设置页低频功能改为图标化折叠入口，默认不展开，减少后台页面纵向占位。
- 新建任务的 AI 辅助入口收进「任务具体需求」标题右侧的小图标，避免把低频智能功能做成大块常驻模块。
- AI 推荐分类不存在时只显示确认动作，不自动写入后台分类库，保持结算相关基础数据可控。
- 已新增统一浮层层级变量，日期 / 月份 / 级联选择器、右键菜单、模态框和覆盖特效不再使用散落的 z-index 数字。
- 任务详情弹窗和任务页右侧详情编辑区已按信息阶段折叠分组，创建 / 排期信息与验收 / 实际进展分开。
- 已完成一次原生日期 / 时间 / 月份控件排查，源码不再使用 `input[type="date"]`、`input[type="time"]`、`input[type="month"]`、`input[type="datetime-local"]`，并补充绿色控件兜底。
- 日期输入统一以 `PlanDateTimeField` 为基座：需要时间时显示小时/分钟列，纯日期场景隐藏时间列；浮层优先留在所属弹窗内，空间不足时允许越界显示，避免裁切关键操作。
- 设置页已改为「业务设置」「权限安全」「系统信息」三组折叠分区，业务设置默认展开，权限和系统低频信息默认收起。
- 新建任务弹窗已增加草稿保护：点击遮罩和 Esc 不会关闭，未创建成功前本地保留草稿，创建成功后清除。
- 设计类型删除、任务时间段删除和验收时间段删除已接入统一确认弹窗，危险删除不再直接执行。
- 管理员登录的 PBKDF2 参数已调整到 Cloudflare Workers 支持范围，避免鉴权异常造成整站无法进入。
- 验收弹窗已增加未保存保护：修改备注 / 时间段 / 上传验收附件后，关闭前必须确认；附件上传中也会拦截关闭。
- 任务详情附件上传中关闭弹窗时会先提示，避免用户失去上传反馈。
- toast、启动加载页、后端状态点和空状态提示已统一，错误提示停留更久并带明确警示图标。
- 新建任务补录月份选择器已压缩为紧凑小面板，低频功能不再占据整行大面积空间。
- 新建任务补录月份进一步改为补录开关旁的一行下拉选择；不再使用 3×4 月份网格或独立面板。
- 作废任务已从浏览器原生 `prompt` 改为站内危险操作弹窗，作废原因在站内表单填写。
- 上传文件过大等错误不再使用浏览器原生 `alert`，改为上传入口下方的站内错误提示。
- 源码已排查 `window.prompt` / `window.alert` / `window.confirm`，后续新增确认、输入、错误反馈必须使用站内组件。
- 作废任务默认从工作台、统计、月报和任务页列表中隐藏；任务页提供「显示作废」开关，作废列表内才允许永久删除。
- 任务右键菜单已补齐查看详情、状态、复制任务名称、复制甲方分享链接、改归属月份、作废 / 删除；文件右键菜单新增下载源文件。
- 任务页「画布模式」调整为「项目文件夹 → 单项目工作流画布」两层结构：第一屏只展示项目文件夹，双击进入项目后才出现该项目的任务、进展、验收和结算节点。每个项目只允许一个任务节点，节点字段对齐 Web UI 的需求人、对接人、验收人、预计开始、预估工时、预计交付、不计费、补录、AI 整理和甲方文案附件，不建立第二套数据入口。

## 后续建议

### P0：优先保证一致性

- 将“确认验收”保留为主按钮，但把“添加时间段 / 上传附件 / 删除时间段”等次级操作继续图标化。
- 删除类操作已统一接入确认弹窗；后续重点转向组件复用和空 / 加载 / 错误状态统一。
- 继续优先排查会造成数据不可见、登录失败、表单丢失、误删误改的稳定性问题；这类问题优先级高于视觉微调。
- 所有包含长文本、上传、验收、付款、结算的弹窗，后续新增时默认要考虑未保存保护或草稿策略。
- 后续新增异步操作时，必须明确区分成功、错误和处理中状态，不要只用一条短暂文案承担所有反馈。
- 低频辅助功能默认应采用紧凑控件或折叠入口，避免在主表单里获得过高视觉权重。
- 单值选择优先用输入 / 下拉形态，只有高频浏览或比较多个日期时才使用展开式网格。
- 禁止新增浏览器原生 `prompt`、`alert`、`confirm`；危险确认用统一站内弹窗，输入原因用站内表单弹窗，错误反馈用 toast 或内联提示。

### P1：节省空间

- 任务详情后续可继续细化时间记录、附件和状态变更的分区文案，但主要折叠结构已经建立。
- 设置页三组 Accordion 已建立，后续只需继续收敛每组内部字段密度。

### P2：组件复用

- 抽出通用 `IconButton`，统一图标按钮尺寸、tooltip、禁用态和危险态。
- 抽出通用 `SegmentTabs`，工作台筛选、任务筛选、详情页签都用同一套。
- 抽出通用 `PanelHeader`，统一标题、副标题、右侧操作区布局。

### P3：右键与快捷键

- 任务右键菜单后续可增加“复制任务名称”“上传附件”“打开文件库”等动作。
- 文件库右键菜单可增加“复制链接”“下载源文件”“移动到归档”。
- 快捷键建议保持轻量：空格预览文件，Esc 关闭弹窗，Enter 保存标签。

## 注意事项

- 涉及结算、验收、删除、状态变更的操作不能只藏在右键菜单里，必须在可见界面中保留入口，避免关键流程不可发现。
- 甲方分享页保持只读，不提供任何删除、改名、标签等管理操作。
- 图标化不是盲目去文字：对低频、风险高、财务相关的操作，仍应保留明确文案或二次确认。
