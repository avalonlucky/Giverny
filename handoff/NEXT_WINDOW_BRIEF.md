## 关键规则

- 正式站：`https://mayeai.com`
- `DEEPSEEK_API_KEY` 已写入正式环境 Worker Secret。
- 正式站只放真实数据；不要在正式站造测试数据、清表或测试上传。
- 发布判断：预发布测试站已彻底下线；本地 lint/build 通过后直接同步正式站。
- 涉及已录入数据、数据库结构、结算口径、文件上传/删除、验收流程、权限等真实业务数据风险时，先在本地或临时隔离环境验证清楚，再部署正式站。
- 任务不允许直接删除，普通任务危险操作是「作废任务」；作废后不出现在工作台、日历、统计、月报、收入和甲方预览中。任务页默认隐藏作废任务，可用「显示作废」开关查看。
- 永久删除只允许在已作废任务上触发，并且必须走站内二次确认；文件库关联文件保留。
- 新建任务不展示阶段选择，也不直接操作验收；已完成补录任务先创建，再到任务详情「确认验收」里补实际工时、验收备注和验收文件。
- 补录月份选择不要做绝对定位悬浮卡片；必须以内联面板占位展示，防止遮挡新建任务表单。
- 不要使用浏览器原生 `input[type="month"]`；月份选择必须用站内自定义绿色选择器。
- 不要使用浏览器原生 `input[type="time"]`；分段计时和验收时间必须用站内 `HH:mm` 文本输入。
- 不要使用浏览器原生 `prompt`、`alert`、`confirm`；危险操作用站内确认弹窗，输入原因用站内表单弹窗，错误用 toast 或内联提示。
- 甲方查看从结算页进入时必须保留「返回结算」入口，避免导航失联。
- 任务状态图例、日历任务点和状态徽章是业务信息编码，必须保留多色区分；不要把计划中紫色、进行中蓝色误判为需要清理的异常品牌色。
- 预计开始时间、预估工时、预计交付时间只用于排期计划；新建任务通过两个小开关选择开始 / 交付锚点进行双向推算，实际工时只来自时间记录 / 验收流程，不要用计划字段参与本月洞察、统计、收入、月报、结算或结算月份兜底。
- DeepSeek AI 助手只给建议，不直接写库；`DEEPSEEK_API_KEY` 必须配置在 Cloudflare Worker Secret 中。AI 文案必须输出三段：`1、设计背景`、`2、设计要求`、`3、输出文件`；把口语表达改成专业任务单语言，但不能凭空编造用户没给的信息。
- 文件可删除，但只用于误传文件，必须二次确认。
- 甲方只通过 `/share/:token` 只读查看月报和文件。
- 每次更新必须同步版本号、`CHANGELOG.md` 和必要的说明文档。

## UI 偏好

- 极简，不要多余容器、阴影和“框套框”。
- 熟悉操作尽量用图标，配 `title` / `aria-label`。
- 可折叠的内容尽量折叠。
- 工作台优先展示任务、待办和关键指标；图表、趋势、年度统计等分析内容默认收进「本月洞察」。
- CSS 颜色和容器规范逐步迁移到基础变量，不要一次性大面积重写稳定页面。
- 浮层层级使用 CSS 变量：普通浮层 `--z-floating`，日期 / 月份 / 级联选择器 `--z-popover`，模态框 `--z-modal`，覆盖特效 `--z-overlay-effect`。
- 任务详情维护要按信息阶段分组：创建 / 排期信息和验收 / 实际进展分开，低频附件和时间轴默认可折叠。
- 设置页使用三组分区：业务设置、权限安全、系统信息；业务设置默认展开，权限和系统低频信息默认收起。
- 新建任务弹窗禁止点击遮罩或按 Esc 关闭；未创建成功前要保留本地草稿，创建成功后才清除草稿。
- 验收弹窗已做未保存保护；修改备注 / 时间段 / 上传附件后，关闭前必须确认。后续新增长表单弹窗也要默认考虑未保存保护或草稿缓存。
- 新增异步操作时必须区分成功 / 错误 / 信息状态；错误提示需要停留更久，避免失败原因一闪而过。
- 补录是低频功能，月份选择器必须保持紧凑，不要再恢复成横向撑满的新建面板大块区域。
- 补录月份是单值选择，必须使用一行下拉 / 输入形态；不要再在新建任务顶部渲染 `CompactMonthSelector` 网格。
- 危险删除操作必须走统一确认弹窗，尤其是文件、口令、设计类型和实际工时时间段；不要新增直接删除按钮。
- 作废、终止、挂起等需要填写原因的流程必须使用站内弹窗，不要再调用浏览器原生输入框。
- 登录和鉴权失败属于 P0 体验问题；如果出现后端异常，先查 Worker 运行时限制和 `/api/auth/login`，不要只看前端样式。
- 次级动作可放右键菜单，但财务、验收等关键动作必须有显性入口。任务右键菜单结构：查看详情、状态二级菜单、复制任务名称、复制甲方分享链接、改归属月份、底部危险操作；挂起 / 终止必须先填原因。
- 不要恢复「甲方可见」标签，甲方默认可看分享页内文件。
