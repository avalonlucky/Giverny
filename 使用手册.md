## 一、这是什么

一个给兼职设计师用的工时管理后台：记录每个月的设计任务、工时、进展和交付文件，月底自动生成银行回执单样式的结算单，一键生成只读链接发给甲方对账。

数据存在 Cloudflare（D1 数据库 + R2 文件存储），换电脑、清浏览器缓存都不会丢。

正式站 `mayeai.com` 只放真实运营数据。预发布测试站及其 D1/R2 资源已删除，功能改版完成本地验证后直接同步正式站。

最新说明：AI 工时建议升级为多维学习模式，会展示任务复杂度画像、历史样本匹配原因、可解释工时拆分、需求人校准和待确认问题；完整原理见「新建任务工时建议」。

最新说明：爱丽丝新增云端历史和 Agent 任务中心。旧浏览器对话会自动迁移，换设备也能恢复完整上下文；后台分析完成后会保留未读提醒，查看结果后才标记已读。

最新说明：爱丽丝支持持久化月度复盘、周工作摘要、风险扫描、跨任务专题、批量附件汇总和多月趋势分析。周一会主动生成周摘要，每月 1 日复盘上月工作；有逾期任务时会主动生成风险提示，结果统一进入任务中心。


最新说明：记录验收进展右侧的「预计工时」会按预计开始与预计交付之间的分钟差精确显示，不再强制按 0.5 小时取整。例如 17:05 到 17:25 会显示约 0.33h。

最新说明：记录验收进展时，验收备注右侧新增「参考任务详情」按钮。点击后会展开并定位到任务基础信息，可以边参考任务名称、设计类型、需求描述、需求人和预计时间，边填写验收备注。

最新说明：新建任务面板支持直接粘贴甲方图片。复制图片后在新建任务面板里按 `Command + V` / `Ctrl + V`，系统会识别剪贴板图片并自动加入「甲方文案附件」，后续可直接用于 AI 需求分析；普通文字粘贴不受影响。

最新说明：任务页「日历视图」的跨天任务在月视图里会按周合并成连续长条。同一周内不再每天重复画短线；如果任务跨到下一周，会在新的一行重新显示任务名称，方便识别跨周任务的持续周期。

最新说明：任务页「日历视图」现在会按任务生命周期连续显示任务条。任务不再只出现在新建 / 开始当天；已验收任务会从开始日期一直显示到验收进展或最后工作段结束日期，未验收任务会用预计交付日期兜底，方便直观看到任务持续天数。

最新说明：任务页「日历视图」的节假日现在独立显示为固定主题色条，使用白色字体；日期头只保留农历小字。休假日显示「休」，补班日显示单独的棕色系「补班」条。

最新说明：任务页「日历视图」新增农历、传统节日、常用纪念日和国务院 2026 年官方休假 / 补班标记。任务条颜色也改成同色系低饱和高亮度浅底，搭配深色文字更柔和。

最新说明：快捷键帮助里的导航组已从 `G + 字母` 改为组合键：`Command + Option + 1~6` 依次前往工作台、任务、文件库、洞察、结算、收入；设置为 `Command + Shift + Option + ,`，知识库为 `Command + Shift + Option + K`。非 Mac 键盘可用 `Ctrl` 替代 `Command`。

最新说明：任务页「日历视图」现在按设计类型大类上色。进入设置页「设计类型」，每个大类都可以绑定一个颜色；任务属于哪个大类，日历里的任务条就显示对应颜色。新增大类会自动使用备选色库中尚未使用的颜色，备选色用完后按顺序循环。

最新说明：右侧栏「记录等待」现在只记录开始时间和备注，不再要求手动填写结束时间。等待时长会自动截止到同一任务里下一段「工作进展分段计时」的开始时间；如果下一段工作进展还没发生，则显示为「等待中」，暂不产生等待时长。

最新说明：工作台顶部「预计收入」脱敏新增快捷键：`Command + Shift + M` / `Ctrl + Shift + M` 可快速显示或隐藏金额，适合旁边有人时迅速切换。

最新说明：工作台 AI 科普会记住当前展示项和预加载队列，并扩大最近标题去重范围；AI 生成到重复标题时会自动重试，减少刷新或换一条时反复出现同一条内容。

最新说明：「洞察」的「总结报告」新增周报化输出：会自动整理完成事项、投入情况和下步计划，并加入“最晚奋斗时间”“最吃工时任务”“深夜仍在线”等盘点式内容。22:00 后或 06:00 前发生的新建任务、验收进展、过程附件 / 验收附件上传会进入「夜间小传记」，方便月底复盘真实工作节奏，也方便把内容直接改写成周报。

最新说明：「洞察」已接入真实交付件理解链路。文件上传到 R2 后，系统会在 D1 创建后台分析任务：图片和 PDF 直接交给视觉模型，PPTX / DOCX / XLSX 提取文字和内嵌图片后分析，结果写回 D1 并在「交付件理解」中展示。分析状态会自动刷新；暂时性失败最多重试 3 次，超大 PDF 可读取首页预览，无法可靠解析的格式会明确提示，不会生成没有依据的结论。

最新说明：工作台右侧任务栏已改为方正矩形面板，顶部只保留任务名称，并在标题下显示新建日期、设计分类和需求人；下方通过「信息 / 进展」两个选项卡切换。右侧栏不再展示附件与动态，过程附件统一在「记录进展」时间轴里查看。

最新说明：右侧栏中的「记录进展」「记录等待」是次级操作入口，已统一为更小的主题绿色文字，避免抢占正文焦点。

最新说明：已修复右侧栏「记录进展」「记录等待」被通用文字按钮样式覆盖的问题，线上会按主题绿色和更小字号显示。

最新说明：右侧栏「可结算 · 段数 · 工时」汇总已改为主题绿色加粗显示，便于快速识别当前可结算工时。

最新说明：右侧栏点击「记录进展」后会打开轻量记录弹窗，只保留进展内容、起止时间和选填附件；进展内容仍保留 AI 优化入口。点击「记录等待」会进入同一弹窗的等待模式，等待时间不计入结算，但会进入后续洞察分析。

最新说明：附件改为「边添加边上传」——在记录进展 / 验收弹窗里选好文件的瞬间就开始后台上传，卡片上有进度条，传完显示「已上传，保存即用」。因此点「保存 / 确认验收通过」时基本秒回，不用再等几十秒。上传后改文件名只改显示名、不会重新上传；删除某个待传附件会一并删掉已上传到后台的文件，不留孤儿。

最新说明：新增「吉维尼模式」（设置 → 外观）。这是一个可选的主题，**默认关闭**，需要在「设置 · 外观 · 吉维尼模式」里手动打开。打开后整站会切换到莫奈花园色系，并且**主题会随季节自然变化**——就像同一片睡莲池在不同时节的水色：春偏嫩绿、夏偏青绿、秋偏暖陶、冬偏水蓝。季节默认跟随当前真实日期，你也可以在设置里手动锁定某一季。不开吉维尼模式时，界面保持原来冷静的工具风格，不受影响。另外，任务明细右边的「开启主题」（任务行随状态上色）现在也默认关闭，需要手动打开。

最新说明：任务行状态配色升级为「方案B」——每种状态有独立的低饱和底色：计划中=中性灰、进行中=淡蓝、待验收=淡米、已验收=淡绿；临期=暖橙、逾期=干净红，底色即可区分状态。该配色现在在「工作台」和「任务」两个页面完全一致；右上角的「关闭主题 / 开启主题」开关是全局的，在任一页面切换都会同步另一页面。

最新说明：「记录进展」中的内容输入和附件区已统一为略深底色；起止时间可手写或通过日历图标选择。系统会按完整的开始、结束日期时间自动计算本段工时，支持跨天，且只有结束时间晚于开始时间时才能保存。

最新说明：右侧栏「记录等待」使用独立弹窗，只记录起止时间和备注；等待原因直接写在备注中。等待时长会进入洞察，但不计入实际工时与结算。

最新说明：记录进展 / 记录等待的开始时间与结束时间之间新增交换按钮，可快速互换两侧时间，适合把某一侧时间复用到另一侧后再手动微调。

## 二、对外介绍功能点

如果甲方或协作方问“这个网站能做什么”，可以按下面这几句话介绍：

- **任务全生命周期**：从计划中、进行中、待验收、已验收到结算，所有状态和关键动作都有记录。
- **真实工时结算**：按分段进展记录实际投入，结算和收入只看真实工时，不靠预计时间凑数。
- **验收闭环**：验收备注、验收附件、进度和工时放在同一个面板里，方便回溯最终确认依据。
- **甲方只读对账**：月底可生成只读结算链接，让甲方查看任务、工时、金额和交付文件，但不能修改后台数据。
- **文件归档**：过程截图、PDF、图片和最终稿跟随任务保存，后续返修、复盘或补交时不用翻聊天记录。
- **洞察复盘**：按周期、项目和需求人分析工时、验收、等待和附件完整度，方便月底复盘和发现异常。
- **AI 辅助**：支持需求整理、进展文案优化、附件理解和洞察复盘，减少人工整理和复盘成本。
- **快捷操作**：支持 `N` 新建任务、`P` 记录进展、`Command + Option + 1~6` 页面导航、`?` 快捷键帮助和 `Command / Ctrl + K` 命令面板，日常操作少依赖鼠标。

## 三、登录规则

打开 mayeai.com 会先看到登录页，有两种登录方式：

### 1. 管理员登录（你自己用）
- **邮箱**：`bh141425@gmail.com`
- **密码**：平台内部管理员密码，可在设置页修改
- 权限：最高权限，比普通用户多一个「口令管理」功能

⚠️ 注意：管理员必须同时输入邮箱和密码，两个都要正确才拥有最高权限。

### 2. 访问口令登录（其他设备 / 协作者用）
- **邮箱留空**，直接输入管理员发放的访问口令（`wk_` 开头）
- 权限：只读浏览为主，不能进行删除等危险操作；管理员才拥有完整管理权限

### 退出登录
左下角「设置」→「退出登录」。登录凭证存在浏览器里，**公共电脑用完务必退出**。

### 登录失效的几种情况
- 口令被管理员停用或删除 → 立即失效，回到登录页
- 口令到期 → 自动失效
- 重新登录即可（管理员账号永远有效）

登录后进入 **设置 → 权限安全 → 账号安全**，填写当前密码和新密码即可。修改后不影响已发放的访问口令。

### 爱丽丝 AI 对话助手（管理员专属）

按 `⌥A`（Option+A）打开/关闭对话弹窗。弹窗居中显示，尺寸为 680px × 80vh。

**界面功能：**
- 首屏欢迎页：大头像 + 「嗨，来和爱丽丝聊一聊」+ 3 个建议提问，点击直接发送
- Header 右侧三个按钮：**新建对话**（将当前对话存入历史并开新对话）、**历史记录**（打开历史列表，可点击恢复，可删除）、**关闭**
- 双击遮罩关闭弹窗
- 输入框为圆角卡片，底部工具栏包含：
  - 📎 附件按钮：支持图片（→ AI 识图）和文本文件（.txt/.md/.json/.csv）
  - 滑块图标：打开「内容范围」选择器，**个人知识库** 和 **全网搜索** 两个开关可独立开关、同时启用
  - 发送按钮；`Enter` 发送，`Shift+Enter` 换行

**知识库：** 按 `Command + Shift + Option + K` 跳转到「知识库」导航页（仅管理员可见），可新增/编辑/删除个人笔记；AI 对话时开启「知识库」开关后会参考这些笔记回答。

**联网搜索：** 需在 Worker Secret 配置 `TAVILY_API_KEY`。开启「全网搜索」后，AI 对非工作台数据的问题会调用 Tavily 搜索互联网补充信息。

**历史记录：** 以云端 Agent 会话为主、本地缓存为兜底。旧浏览器记录首次打开时会自动迁移；回答、确认卡、分析任务和附件结果可跨设备恢复。

**Agent 内核：** 爱丽丝基于 Cloudflare Agents SDK 运行，每个对话拥有独立的持久会话。模型先理解问题并选择类型化工具，工具从 D1 读取真实任务、工时和金额，再由模型组织回答。金额、工时、月份等数字严格来自工具返回；执行期间会沿时间线逐步显示理解、规划、工具调用、结果返回和回答整理等可验证动作，完成后自动折叠为「执行记录」，不展示模型内部思维链。

询问“某个项目有哪些附件”“帮我找邀请函并打开”等问题时，Agent 会调用附件搜索工具。回答正文中的标题、列表和表格会按正式富文本排版；真实文件显示为独立附件卡，可查看所属任务、格式、大小和标签，并直接预览或打开。文件地址由工具返回和权限接口校验，不由模型在正文里编造。

创建任务、记录反馈、修改字段、修改状态和追加进展都必须先生成确认卡，只有点击确认后才由 Cloudflare Workflow 持久执行。月度复盘等耗时只读分析也使用独立 Workflow：对话中会显示真实进度卡，页面关闭后仍继续，完成时主动通知，可取消或失败后重试。

**外部 Agent 只读工具接口：** Worker 同时通过 `/mcp` 暴露只读 MCP 工具。它使用独立的 `MCP 只读`访问口令，不能登录网站，也不能执行任何写入：

- `GET /api/agent/tools/month-finance`：月份金额统计（总金额、计费工时、时薪、任务明细）
- `GET /api/agent/tools/search-tasks`：任务搜索
- `GET /api/agent/tools/search-attachments`：附件搜索与可验证文件地址
- `GET /api/agent/tools/task-detail`：任务详情
- `GET /api/agent/tools/context`：爱丽丝能力边界说明
- OpenAPI 描述：`/api/agent/openapi.json`；支持 MCP 的客户端优先使用 `https://mayeai.com/mcp`。

### 多模态模型兜底
设置页「AI 模型设置」现在支持四路配置：文字主模型、文字备用模型、识图主模型、识图备用模型。

默认策略：
- 文字任务：DeepSeek 优先，Kimi K2.6 兜底
- 识图任务：Gemini 3 Flash 优先，Kimi K2.6 兜底

面板按「文字模型 / 识图模型」两个选项卡分类，切到哪类就只看该类的主力与备用两张卡。每一路都可以配置供应商、Base URL、模型名称和 API Key，并可以直接在设置页点击「测试」确认是否可用。平台默认 Key 建议写入 Cloudflare Secret，例如 `GEMINI_API_KEY`、`KIMI_API_KEY`；多租户后也可以让租户在设置页自行填写并加密保存。

操作上的几个便利：
- **供应商联动 Base URL**：选择供应商时，Base URL 会自动填好——DeepSeek / Gemini / OpenAI 默认指向 Cloudflare AI Gateway 对应路径，Kimi / OpenRouter / Anthropic 走官方直连，并带出该供应商默认模型名，需要时再手改即可。
- **「改走网关 / 改为直连」一键切换**：网关支持的供应商可在网关路径与官方直连地址间一键互换，点「测试」即可分别验证哪种可用，再也不用猜。
- **「获取模型」按钮**：点一下实时拉取该供应商最新可用模型，输入框下方出现原生下拉，换模型直接选。拉取使用该路由已保存或环境变量里的 Key，仅管理员可用。

### Workers AI 终极兜底
平台启用了 **Cloudflare Workers AI**（边缘自带开源模型，无需任何外部厂商 Key），作为「文字主模型 → 文字备用模型 → Workers AI」三层链路的最后一道兜底——即便 DeepSeek、Gemini、Kimi 全部不可用，AI 功能也不会完全瘫痪。默认模型为 Llama 3.3 70B，可经环境变量 `WORKERS_AI_MODEL` 覆盖。计费按 Neurons（$0.011/1000 Neurons，每天免费 1 万 Neurons）。

> 注意：Workers AI 提供的是 Cloudflare 自托管的开源模型（Llama、Qwen、Mistral 等），**不包含 GPT、Gemini 这类闭源商业模型**——后者仍需用各自厂商的 Key，经 AI Gateway 代理调用。

### OpenRouter 免费模型每日实测
设置 → AI 模型设置 → 「OpenRouter 免费模型」面板：系统每天自动拉取 OpenRouter 上全部 `:free` 免费模型并**逐个实测**，标出真实可用的（可用/限流/已下架），并区分文字/识图与上下文长度。可点「立即扫描」手动刷新；可用模型可一键「设为文字备用 / 设为识图备用」，再点上方「保存 AI 设置」即生效。免费模型有速率上限，建议只作兜底备用。需先配置 `OPENROUTER_API_KEY`（Worker secret）。

### 结算回单高清 PDF
「结算」页的「导出 PDF」走 Cloudflare Browser Rendering 服务端无头浏览器渲染，输出**清晰矢量 PDF**（A4：抬头、客户信息、任务表、合计、人民币大写、本月不计时、备注），不再是浏览器端截图那种可能模糊/错位的效果。仅管理员可用；服务端不可用时会自动回退浏览器打印。

### 历史任务语义搜索
顶栏的「语义搜索」（放大镜）按意思找回历史任务：输入"之前那张邀请函长图""官网 banner"之类自然语言即可，按匹配度排名，点击直达任务详情。底层用 Cloudflare Vectorize（向量库）+ Workers AI `bge-m3` 多语向量，把任务标题/类型/需求/验收备注入库；新建或编辑任务后会自动更新索引（约 1 分钟生效），管理员可在弹窗里「重建索引」做全量重建。仅管理员可用。

### BAML 与多模型路由
项目已引入 BAML 工程结构，AI 提示词和输出类型会先维护在 `baml_src/ai_assistants.baml` 中，并通过 `npm run baml:generate` 生成 TypeScript client。

正式站会优先调用独立 BAML Runtime，并在 Runtime 不可用时回退到 Cloudflare Worker 内的文字主模型 / 备用模型。模型负责理解和解释，任务、工时、附件和结算数据仍以 D1 / R2 的真实记录为准。

### 新建任务工时建议
新建任务时，在「预计开始时间 / 预估工时 / 预计交付时间」下方可以点击「AI 分析」获取工时建议。

建议结果会展示：
- 常规预估工时和稳妥预留工时
- 置信度与任务复杂度分数
- 精确同类样本数和相关参考数
- 历史中位工时、范围和参考任务
- 复杂度画像、可解释工时拆分和主要分析依据
- 需求人历史校准情况
- 仍需确认的问题，以及每个参考任务为什么相似

#### 1. 任务复杂度画像

系统不会只按「海报」「PPT」等关键词判断，而会先把当前任务拆成统一的复杂度画像：

- **设计基础**：从零设计，还是复用已有方案、上一版或同系列风格。
- **交付规模**：需要多少张、多少页、多少个版本或交付单元。
- **内容准备**：文案、图片和品牌素材是否齐全，是否还需要整理、提炼或补写。
- **版本适配**：是否包含横竖版、多尺寸、多平台、多端或多个比例。
- **专项处理**：是否涉及插画、图表 / 信息可视化、精修合成、三维、动效视频或复杂文档排版。
- **协作风险**：是否加急、方向未定、多人确认或预期存在多轮改稿。

这些维度会生成 0–100 的复杂度分数，并归为低 / 中 / 高复杂度。每个维度都会显示从需求中提取到的依据，方便人工判断系统是否理解正确。

#### 2. 分层检索历史样本

系统只把已验收并有真实工时的任务作为权威样本，再分层检索：

1. **同一设计类型 + 复杂度画像相近**：优先级最高，例如同为「传播类 / 海报」，且同样是从零设计、单张交付、文案齐全。
2. **需求语义相似 + 工作量维度接近**：通过向量语义检索找到“意思相近”的历史任务，再比较交付数量、适配、专项处理等维度。
3. **同一大类参考**：只有前两层样本不足时，才少量参考同大类任务，并使用更低权重。
4. **同一需求人**：类型和复杂度接近时，同一需求人的历史任务会获得小幅加权，但不会压过真实任务相似度。

参考任务列表会说明“同一设计类型、需求语义接近、复杂度画像接近、同一需求人、同为复用型任务”等匹配原因，不再只显示一个模糊的相似度。

#### 3. 可解释的工时拆分

AI 不只回答“预计 4 小时”，还会把常规预估拆成可核对的模块，例如：

| 模块 | 示例工时 | 说明 |
|---|---:|---|
| 核心设计 | 2.5 h | 从零搭建设计方向、信息层级和主版式 |
| 批量制作 | 0.6 h | 多张物料或多页内容的延展与一致性检查 |
| 内容整理 | 0.3 h | 核对文案、图片和版面结构 |
| 版本适配 | 0.8 h | 3 个尺寸 / 平台的版式适配 |
| 沟通与改稿 | 0.3 h | 常规对接和一轮调整 |
| 交付整理 | 0.1 h | 文件检查、命名、导出和交付复核 |

实际模块会根据本次任务画像变化。拆分结果用于解释建议，不会自动生成实际工时；实际结算仍只读取保存后的分段计时。

#### 4. 需求人校准

当某个需求人积累至少 3 条“预估工时 + 已验收实际工时”的完整样本后，系统会建立受控校准：

- 历史实际投入通常高于还是低于原预估。
- 平均改稿轮次。
- 历史任务需求完整率。
- 当前任务与该需求人历史任务的类型和复杂度是否接近。

独立系数会限制在合理范围内，避免少量异常项目把结果拉偏。样本不足 3 条时只展示已有观察，不调整建议值。

#### 5. 持续学习的完整生命周期

```text
当前任务需求和附件
  → 复杂度画像与历史样本检索
    → 常规 / 稳妥工时建议
      → 用户采用或手工修改
        → 任务实际分段计时
          → 最终验收实际工时
            → 回写同类型偏差与需求人校准
              → 下一次建议更准确
```

系统会记录当次输入指纹、建议值、最终选定值、分析依据和验收后的真实工时。学习只用于后续检索、排序和数值校准，不允许模型自行修改历史任务、实际工时、结算金额、提示词或权限。

#### 6. 如何获得更准确的建议

- 尽量写清从零还是复用。
- 写清交付数量、页数、尺寸和平台。
- 说明文案素材是否齐全。
- 说明是否有插画、精修、三维、动效等专项处理。
- 说明确认方和预计修改轮次。
- 上传甲方需求附件后重新点击「AI 分析」。

系统只使用已验收任务的真实工时作为权威结果。预计开始到交付之间的自然时间差只用于排期背景，不会被当作实际工作量；没有可靠历史样本时保留当前手工预估。

点击「采用常规值」或「采用稳妥值」后，系统会把对应工时写入「预估工时」，并按当前排期锚点自动推算预计开始或预计交付时间。任务信息或需求附件变化后，旧建议会失效，需要重新分析。

更完整的产品和技术口径见 `docs/AI_HOUR_ESTIMATE.md`。

## 五、日常使用

### 顶栏（所有页面通用）
- **月份切换**：所有页面的数据都跟着所选结算月份走；月份选项根据任务的结算月份自动生成，永远包含当前月
- **结算**：从左侧导航进入结算页，核对月度结算单、锁定结算并生成甲方链接
- **新建任务**：随时可建
- **状态反馈**：操作成功、失败和处理中会用不同的提示样式；侧边栏标题下方的小圆点表示后端连接状态，异常时会变红。

### 工作台
当月总览：总工时、计费工时、预计收入、验收情况四张统计卡；任务明细列表（搜索框 + 筛选下拉，可按计划中/进行中/待验收/已验收过滤）；设计类型工时分布环形图、每周工时趋势图、年度统计柱状图都只统计实际工时，预计开始时间、预计交付时间和预估工时只作为排期参考。

### 任务
- **列表视图 / 日历视图**：进入任务页后，在页面内容区切换，不在左侧导航栏里展开。
- **列表视图**：任务列表会直接展示任务名称、需求摘要、对接人、实际工时、状态、进度、预计开始、预计交付和排期倒计时；搜索框在标题行右侧，状态筛选和「显示作废」在下一行，方便快速扫读。
- **排期提示**：任务行会根据预计开始和预计交付自动显示「距开始」「距交付」「今日 / 明日到期」「已逾期」或「已验收」。列表底部有「临期 / 逾期 / 进行中 / 正常」图例说明颜色含义。这些时间只用于排期提醒，不参与实际工时、收入统计、月报或最终结算。
- **信息 / 进展**：右侧详情区包含两个选项卡。信息页看基础字段；进展页按时间轴记录任务从接受、状态变化、时间段记录、文件上传到验收的全过程。
- **查看任务详情**：详情弹窗用于快速核对任务字段、进度和最近动态；右上角「待验收」状态悬停后会变为「去验收」，点击即可打开验收面板。底部只保留「进展」和「去编辑」两个可操作入口，关闭使用右上角图标。
- **弹窗关闭规则**：所有弹窗默认不会因为点击外部空白处或按 Esc 关闭，避免误点丢内容；需要关闭时点击右上角关闭图标或弹窗里的取消按钮。新建任务、记录进展和确认验收会保留未保存草稿，保存 / 确认后才清空。
- **右键快捷操作**：在工作台任务行或任务页列表中右键任务，可快速查看详情、记录进展、去验收、调整进度或复制已锁定月份的甲方分享链接；「状态」会以二级菜单收纳进行中、待验收、挂起和终止。挂起 / 终止会要求填写原因。
- **空白处右键新建**：在工作台任务列表或任务页列表空白处右键，可以直接打开「新建任务」菜单；未登录管理员时会先提示登录。
- **AI 优化进展 / 验收备注**：在「记录进展」弹窗的新增进展标题右侧、以及「确认验收」弹窗的验收备注标题右侧，都有 AI 图标。点击后，系统会结合当前填写内容、任务需求、任务相关文件、过程附件 / 验收附件和最近进展生成建议；建议需要手动点「采用建议」才会写回输入框。
- **AI 持续学习**：系统会对比生成建议前的原始输入、AI 建议和最终保存内容，并记录直接采用、修改后采用或未采用。任务需求、任务名称、分类、进展、修改意见、验收备注和附件命名会分别积累偏好；达到足够样本后，后续建议会优先贴近你的真实写法。学习只影响建议，不会自动修改历史任务、工时、结算或验收状态。
- **进度与分段计时**：「记录进展」弹窗统一记录过程数据，包括整体进度、可结算分段计时、等待记录、进展说明和过程附件。分段计时会汇总为实际工时，用于后续验收和结算。
- **等待记录**：任务进展页的「时间记录」只记录可结算工时；「等待记录」用于记录等待甲方意见、补资料、等确认等非计费占用。等待记录不会进入结算工时，但会进入洞察，用于分析任务从接受到验收期间被项目占用但未产生收入的时间。
- **验收核对**：确认验收弹窗只负责终审核对和锁定结果。基础信息、整体进度、可结算分段计时、等待记录、验收附件与备注会分区展示；进度、分段计时和等待记录都来自「记录进展」，验收里不再临时添加或修改这些过程数据。
- **工时与等待汇总**：验收面板会按日期、时间段、内容和工时列出可结算分段计时，并显示实际总工时、后台时薪和本次预计结算金额；等待记录单独汇总，只用于洞察分析，不进入结算工时。
- **本次计入工时开关**：验收进展面板「时间与工时」标题右侧有「计入工时 / 不计入工时」开关，默认计入。关闭后本次验收不新增工时（开始/工时/结束整体置灰），但之前已汇总的工时原样保留，仍可直接保存或验收——适合临近验收时一两分钟的小改动不想计入总工时的特殊情况。
- **验收主观反馈**：确认验收时可选「顺利 / 一般 / 有问题」，并按需补充「需求不清晰 / 沟通成本高 / 定价偏低 / 技术挑战大」原因标签和体感评价。不填也能验收；这项只给管理员自己看，用于记录客观工时之外的任务体感。
- **删除任务动态**：在「记录进展」弹窗的进展时间轴中，管理员可右键单条动态，或悬停后点击删除图标，删除前会出现站内确认弹窗；删除只移除时间轴记录，不回滚任务字段、文件或工时数据。
- **任务不允许删除**：任务会影响工时、收入和税务统计，正式运营后不做删除。暂时不做的任务选择「挂起」，确定不做的任务选择「终止」并填写原因。
- **作废任务**：测试任务或误录任务可通过右键菜单作废，系统会弹出站内确认框，可填写作废原因；作废后不计入工时、收入和结算，也不出现在工作台、日历和月报里，但保留历史记录和关联文件。
- **查看 / 删除作废任务**：任务页默认隐藏作废任务，可打开「显示作废」查看；只有已作废任务才允许永久删除，删除前仍会二次确认。
- **排序规则**：工作台和任务列表先按“结算月份”筛选，再按“预计开始时间 / 开始日期倒序”排列；同一天内按创建时间倒序，新建的排在更上面。预计开始时间只影响列表排序和排期提醒，不参与统计、工时计算或结算。
- **计划时间**：新建任务时可填写预计开始时间到分钟，例如 `2026/06/13 10:10`。新建面板支持双向推算：打开「预计开始时间」旁的开关时，系统按开始时间 + 预估工时推算交付时间；打开「预计交付时间」旁的开关时，系统按交付时间 - 预估工时倒推开始时间。未打开开关的一侧只读展示。预计开始时间、预计交付时间和预估工时只用于排期参考，不影响本月洞察、实际工时、收入统计、月报或最终结算。
- **AI 优化需求**：新建任务时，「任务具体需求」右上角有 AI 图标。填写项目名称或原始需求后点击，可让 DeepSeek 助手把口语化描述整理成「设计背景 / 设计要求 / 输出文件」三段任务单语言，并推荐设计类型。建议只会展示给管理员，采用文案、采用分类或新增分类都需要手动确认。「甲方文案附件」支持上传图片（JPG/PNG，最多 6 张）和文本文件；上传图片后 AI 会先识图提取完整内容，再与需求文字交叉校对——若你只写了简称（如「融合防勒索与零信任」），AI 会自动从图片中找到完整官方名称并填入设计背景。设计要求只写甲方明确指定的约束，无明确视觉要求时不会自行规定颜色或风格。
- **需求修改会反哺 AI**：生成需求建议后，无论你直接采用、采用后继续修改，还是完全按自己的写法提交，系统都会把最终任务单与当次建议交叉比对。可比较样本积累后，AI 会按设计类型归纳你的删改习惯、三段式写法和任务命名偏好。
- **补录历史任务**：新建任务弹窗右上角有「补录」文字和一个极简小开关，默认关闭。只有特殊补录场景才打开，打开后在开关旁显示「计入月份」下拉，不会占用主表单空间。例如任务实际 5 月完成，但需要放入 6 月工资一起结算，就把预计开始时间按真实 5 月填写，同时把计入月份选为 6 月。这样它会出现在 6 月工作台、任务列表和 6 月月度结算单中，并在日期旁标记“补录”。
- **结算月份**：普通新任务默认计入顶部当前月份；修改预计开始时间或预计交付时间不会自动改结算月份，需要调整归属时请显式修改「结算月份」。
- **补录状态建议**：新建任务不选择阶段，也不直接操作验收。已经完成、只需要补结算的任务，创建时填写必要的「补录说明」，创建后进入任务详情点击「确认验收」，再补实际工时、验收备注和验收文件。还没开始或正在做的任务，创建后按真实进度在任务详情或任务列表中调整状态。

### 工作台右侧任务明细卡
在工作台的任务明细里**点任意一行**，右侧明细卡同步显示该任务：
- 任务状态（徽章）、**预计交付时间**（临期/逾期会标黄/标红）、实际工时、对接人、验收人
- 整体进展条
- **上传附件**入口：选完文件自动上传并关联到这个任务
- **动态时间轴**：最近 4 条操作记录
- 底部**「查看更多」**按钮 → 展开完整详情弹窗

### 任务详情弹窗（点「查看更多」展开）
- 展示任务基础字段、预计开始、预计交付、任务状态、当前进度、实际工时、结算月份和最近进展。
- 不再提供单独文件上传入口。过程附件从「进展」里上传；验收附件在「去验收」终审面板里补传。
- 待验收任务右上角状态可直接进入验收，确认前仍会核对工时、备注和验收附件。
- **最近动态**：展示最近操作记录；完整过程记录请进入「进展」查看。

### 交付日期提醒
- 新建/编辑任务时填「预计交付时间」，系统自动盯着：
  - **临期**（3 天内要交付）：黄色标记
  - **已逾期**（过了交付日还没验收）：红色标记
- 工作台顶部会出现黄色提醒条（统计当前月份的临期/逾期任务），点击直接跳到任务页
- 已验收和不计费的任务不参与提醒

### 任务状态规则
| 状态 | 含义 | 计费 |
|------|------|------|
| 计划中 | 已立项还没开工（新建任务默认） | ❌ 不进结算单 |
| 进行中 | 正在做 | ✅ |
| 挂起 | 暂时暂停，必须填写挂起原因 | ❌ 暂不计入结算 |
| 待验收 | 做完等甲方确认 | ✅ |
| 已验收 | 甲方确认完成 | ✅ |
| 终止 | 需求取消或确定不继续，必须填写终止原因 | ❌ 不进结算单 |
| 不计费 | 帮忙性质，不收钱（免费协助） | ❌ 不计入计费工时与收入，但以 ¥0 行出现在结算回单 |

> 「不计费任务」可在**新建任务**弹窗右上角打开「不计费」开关直接创建：整条任务从创建到交付都不计费，不计入计费工时和收入，但仍会以 ¥0 行体现在结算回单里（标注「不计费 / 免费协助」），合计金额不受影响。适合人情或极简单的免费帮忙。

### 进展
- 进展已合并到任务详情里，不再作为左侧独立导航。
- 可以记录某个时间段做了什么，例如 `14:00-15:30 修改主视觉文案层级`。这些时间段会纳入任务实际工时。
- 状态变化、接受任务、上传文件、确认验收等操作也会写入时间轴。
- 确认验收时，如果已经修改验收备注、调整时间段或上传验收文件但还没点「确认验收」，关闭弹窗前会先提示确认；附件上传中或系统计算工时为 0 时不能提交验收。

### 文件库
- 文件库不是单独上传入口，而是自动汇总任务生命周期中的文件。
- 文件来源包括：任务过程文件、进展附件、验收时上传的最终稿和验收文件。
- 文件按“项目 / 任务”归档，点击某个任务后查看该任务下所有关联文件。
- 支持搜索文件名、任务名和标签；支持按月份筛选。
- 点击文件卡片会打开右侧信息栏，可重命名、添加多个标签、预览、打开或删除。
- 右键点击文件卡片可快速预览、打开、重命名、添加标签、下载源文件或删除；重命名和添加标签会自动聚焦右侧信息栏。
- 删除文件只用于处理误传文件。点击删除后会弹出站内二次确认框，确认后才会删除 D1 文件记录、R2 源文件和预览图。
- 已验收、已发给甲方或可能用于回溯的文件建议保留。
- 空格键可以快速预览当前选中文件。
- PSD、AI、PDF、Word、Excel、PPT、MP4 和图片尽量提供在线预览；部分源文件格式依赖浏览器和预览图能力，必要时仍可打开或下载源文件。

### 洞察（仅管理员可见）
- 工作台和任务管理列表会在部分任务行旁显示 **上下文洞察**，例如同类任务历史平均超时、当前预估低于同类均值、同类修改信号偏高等。它只在历史样本足够且信号明显时出现，悬停可查看依据。
- 左侧导航进入 **「洞察」**（仅管理员，游客 / 甲方成员会看到「仅管理员」提示面板），含周期复盘、交付件理解、需求人画像等分区。
- 「交付件理解」展示附件的分析状态、模型摘要、需求匹配、质量问题、风险与建议。历史文件可点击「补分析历史附件」，失败文件可单独重试。
- 图片和 18MB 以内 PDF 可直接识别；35MB 以内 PPTX / DOCX / XLSX 当前基于文档文字和内嵌图片分析。PSD / AI 需要上传预览图，压缩包和旧版 Office 文件暂不自动解析。
- 「周期分析」可以按日、周、月、季度、半年和年度切换分析周期，查看周期任务数、验收率、实际工时、预估工时偏差、交付件数量、类型工时结构和进展投入趋势。
- 「交付件分析」会把任务、进展、上传文件和验收状态串起来，点击 PDF、JPG、PPT 等文件标签可以直接打开预览。
- 「能力画像」会展示设计类型覆盖、工时估算准确率、平均交付周期和对接人集中度，帮助判断效率稳定性和客户来源是否过度集中。
- 「需求人画像」是对**这个需求人本身**的多维考察（交付文件内容洞察属于「交付件理解」，不在此处）。核心指标含合作项目、计费工时、验收通过率、单项目均时（与你其他需求人均值对比）、准时交付率、平均工时偏差、平均改稿轮次、等待耗时；并把这些数据翻译成对人的特征判断，例如「验收通过率高，多数一次过」「需求表达明确，改稿少」「单项目耗时偏长/偏短」「需求相对简单、好交付，较容易获得工时」「确认及时/偏慢」「工时可预估，报价风险低」「改稿轮次偏多/利落」「延迟率偏高/几乎不拖期」等，按好/一般/注意三色标注；并汇总验收时的体感反馈（顺利/一般/有问题分布与高频反馈标签），最后据此给出多条可执行的报价/排期/协作建议。
- 附件分析默认优先使用 Gemini 3 Flash，失败时切换 Kimi K2.6；文字助手仍优先使用 DeepSeek，失败时使用 Kimi K2.6。
- 后台每 5 分钟会继续处理 1 个未完成分析，单个失败任务最多自动尝试 3 次；升级 Cloudflare Paid plan 后可再提高单次处理量。
- 「追踪中的洞察」会保留每个异常触发项的最新发现、建议、当时数据和状态。后台每 5 分钟用规则扫描异常，命中后再调用 AI 更新专项建议；异常消失时自动结案，不会反复堆积同一条洞察。（页面顶部不再提供「历史视角」「运行诊断」手动按钮。）
- AI 洞察会读取验收主观反馈和原因标签，用来识别“工时正常但做得痛苦”的任务模式，例如某个需求人近期集中出现「沟通成本高」。
- AI 洞察会读取等待记录和任务周期，分析总周期小时、周期占用等待、明确等待小时和等待占比。等待占比高代表任务拖得久但计费少，可能挤占本可以接其他活的时间。
- 当前事件触发包括：同类任务修改信号偏高、本月工时比上月下降超过 20%、某个对接人综合时薪连续低于均值、某类设计类型超过 3 个月没有新任务、同类任务等待占比偏高。
- 修改轮次尚未单独录入时，系统采用进展文字中的“修改、调整、改稿、反馈、返工”等 **修改信号** 作为代理，不会把它伪装成精确轮次。

### 结算（月报核心流程）
月底结算的标准流程：

1. 切到要结算的月份，核对回执单上的任务和工时
2. 点 **「锁定结算并生成甲方链接」** → 金额快照存档，链接自动复制
3. 把链接（`mayeai.com/share/xxxx`）发给陈义君
4. 需要 PDF 的话点「导出 PDF」→ 打印窗口选「另存为 PDF」（文件名自动带月份）

**锁定规则：**
- 每个月只有一条结算记录；锁定后又改了数据，再点一次锁定即可更新金额，**链接不变**（已发出的链接不会失效）
- 锁定的金额是快照——以后改时薪不影响已锁定的月份
- 「结算历史」列出所有锁定过的月份，随时复制链接或打开查看
- **甲方查看回执**：甲方打开过链接后，结算历史会显示「甲方已查看 N 次（最近时间）」；没打开过显示「甲方尚未查看」——月底催验收前先看一眼

### 年度统计（工作台底部）
- 全年 12 个月的计费工时柱状图，🔒 表示该月已锁定结算
- 累计工时、累计收入一目了然（已锁定月份按结算快照计，未锁定月份按当前时薪实时算）
- 鼠标悬停柱子可看每月明细

### 甲方分享页（甲方看到的东西）
- 不需要任何口令，凭链接直接看
- 内容：锁定的工时与金额、任务明细（含「不计费」任务，以 ¥0 行单独列出）、该月份关联的进展和文件
- 文件点击弹窗在线预览；页面可直接打印成 PDF
- 甲方只能看到分享页内容，不能进入后台，也不能新增、修改或删除任何数据

### 结算回单 PDF
- 「结算」页下方的回单可通过「导出 PDF」打印保存。
- 回单包含结算摘要表和「工时明细附表」。
- 「工时明细附表」字段参考 Excel「6月」工时明细表结构，包括参考开始日期、设计类型、任务名称、需求、对接人、阶段、参考预估工时、实际工时、参考交付日期、实际交付日期、状态、验收人、风险/阻塞和进展；合计与金额只看实际工时。
- 明细内容来自平台任务、验收备注、时间记录和进展记录，不会直接复制 Excel 里的旧数据。

### 设置
- 左下角「设置」是管理员菜单，包含全站设置入口、R2 文件空间、退出登录和当前版本号。
- 点击左下角菜单里的「全站设置」进入完整设置页。
- **小时单价**：改了之后预计收入自动重算（已锁定的月份不受影响），默认 ¥300/小时
- **服务公司名称**：用于月度结算回单和分享查看页，可在设置页直接输入修改。

## 六、计费规则速查

- 预计收入 = 计费工时 × 小时单价
- 计费工时 = 当月所有任务实际工时之和，**只排除「不计费」任务**；任务状态（计划中 / 进行中 / 待验收 / 已验收）不影响计费，计费完全以「计时与工时汇总」记录的实际工时为准
- 结算金额 = 每条计费任务「实际工时 × 时薪」按**真实金额**计入（保留小数、精确到分，不四舍五入到元），再求和；保证回单「明细金额之和 === 总额」可逐行对账，绝不出现“差几元对不上”
- 回单明细只列出「有实际工时」的计费任务（工时为 0 不入账）；「计划中」数量在备注中单独提示
- 统计只读取 D1 中的正式任务数据和已锁定月报快照，不再包含任何硬编码导入工时
- 结算单序号、人民币大写金额、电子印章均自动生成

## 七、运营与数据安全

### 正式站
- 正式站：`https://mayeai.com`，只录入真实任务、真实工时和真实交付文件。
- 预发布测试站已下线，测试 D1/R2 资源也已删除。
- 每次功能改动完成本地验证后直接同步正式站。

### 文件库清理原则
- 当前没有自动按月删除机制，避免误删当月文件或影响甲方月报。
- 推荐每月结算后执行一次人工复核：确认月报已锁定、甲方已验收、备份已导出，再考虑清理低价值过程文件。
- 最保险的做法不是直接删除，而是后续增加“归档 / 冷存储 / 待删除队列”机制，给文件至少 30 天反悔期。
- 最终稿、验收文件、已发给甲方的文件不建议删除；源文件是否删除应以“是否仍可能返修 / 补交”为准。
