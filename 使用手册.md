## 一、这是什么

一个给兼职设计师用的工时管理后台：记录每个月的设计任务、工时、进展和交付文件，月底自动生成银行回执单样式的结算单，一键生成只读链接发给甲方对账。

数据存在 Cloudflare（D1 数据库 + R2 文件存储），换电脑、清浏览器缓存都不会丢。

正式站 `mayeai.com` 只放真实运营数据。预发布测试站及其 D1/R2 资源已删除，功能改版完成本地验证后直接同步正式站。

最新说明：「洞察」已接入真实交付件理解链路。文件上传到 R2 后，系统会在 D1 创建后台分析任务：图片和 PDF 直接交给视觉模型，PPTX / DOCX / XLSX 提取文字和内嵌图片后分析，结果写回 D1 并在「交付件理解」中展示。无法可靠解析的格式会明确提示，不会生成没有依据的结论。

最新说明：工作台右侧任务栏已改为方正矩形面板，顶部只保留任务名称，并在标题下显示新建日期、设计分类和需求人；下方通过「信息 / 进展」两个选项卡切换。右侧栏不再展示附件与动态，过程附件统一在「记录进展」时间轴里查看。

最新说明：右侧栏中的「记录进展」「记录等待」是次级操作入口，已统一为更小的主题绿色文字，避免抢占正文焦点。

最新说明：已修复右侧栏「记录进展」「记录等待」被通用文字按钮样式覆盖的问题，线上会按主题绿色和更小字号显示。

最新说明：右侧栏「可结算 · 段数 · 工时」汇总已改为主题绿色加粗显示，便于快速识别当前可结算工时。

最新说明：右侧栏点击「记录进展」后会打开轻量记录弹窗，只保留进展内容、起止时间和选填附件；进展内容仍保留 AI 优化入口。点击「记录等待」会进入同一弹窗的等待模式，等待时间不计入结算，但会进入后续洞察分析。

最新说明：「记录进展」中的内容输入和附件区已统一为略深底色；起止时间可手写或通过日历图标选择。系统会按完整的开始、结束日期时间自动计算本段工时，支持跨天，且只有结束时间晚于开始时间时才能保存。

最新说明：右侧栏「记录等待」使用独立弹窗，只记录起止时间和备注；等待原因直接写在备注中。等待时长会进入洞察，但不计入实际工时与结算。

最新说明：右侧栏「进展」下新增「工作进展分段计时 / 等待记录不计结算」二级切换，默认只展示最新 5 条记录，超过 5 条时通过展开按钮查看全部。进展和等待记录均可二次编辑；同类时间段不能互相重叠，但首尾相接的连续记录可以保存。

最新说明：记录进展 / 记录等待的开始时间与结束时间之间新增交换按钮，可快速互换两侧时间，适合把某一侧时间复用到另一侧后再手动微调。

最新说明：记录进展 / 记录等待的时间交换按钮现在只负责左右互换开始和结束时间；如果互换后结束时间早于开始时间，保存按钮会保持不可用，需要手动改好结束时间。新建记录默认读取打开弹窗时的本地当前时间，并在重叠时提供最近空档的一键切换建议。

最新说明：工作台右侧详情面板的进展滚动条已改为浅绿色细条，降低深色原生滚动条对内容的干扰；滚动到底后继续滚动仍会接力到整页。

最新说明：右侧栏「工作进展分段计时」和「等待记录不计结算」均按开始时间倒序展示，最新时间段在上方，跨天记录也按开始时间参与排序。

最新说明：确认验收面板已改为更接近终审确认单的排版：顶部留白更大，基础信息三列展示，计时与工时汇总采用正式表格，等待记录单独显示为洞察数据；验收备注仍保留 AI 优化入口。

最新说明：日期时间选择器在同一个弹窗里只会同时打开一个；选中日期、小时或分钟后不会自动关闭，方便继续调整，确认后可点击其他字段或关闭按钮收起。

最新说明：新建任务弹窗已按轻量参考稿重做，采用方正窄面板、胶囊分类、底线输入和浅底需求框；补录、AI 需求优化、AI 工时建议及草稿保存能力继续保留。

最新说明：新建任务弹窗已进一步按原型 HTML 精确校正：设计类型使用父分类胶囊和悬浮深色子菜单，补录月份与开关独立展示；时间与工时同时启用两项，第三项自动推算并灰显。

最新说明：新建任务弹窗已修复补录头部换行、分类文字颜色与字重、日期输入绿线和操作按钮过重的问题，整体对齐原型的轻量排版。

最新说明：补录开启后，记录月份与补录开关会保持同一行横向展示；弹窗已加宽以避免中文按钮被挤成竖排。

最新说明：补录月份只允许选择当前月及向前 3 个月，不会出现未来月份或过久历史月份。

最新说明：全站日期、日期时间与月份选择器统一复用同一套年月面板。点击日历图标后，可通过顶部上下箭头切换前后月份；点击年月可直接选择年份和月份；需要时间时显示右侧小时、分钟列，仅日期时自动隐藏；底部提供「清除」与「今天」，仍支持手写输入。

最新说明：任务整体进度统一按 20% 档位记录和展示。工作台右侧栏、记录进展弹窗、右键快捷菜单和状态流转都只会使用 0 / 20 / 40 / 60 / 80 / 100。

最新说明：动态时间轴的具体到分钟时间仅管理员可见；普通成员、甲方预览和公开只读报告只显示日期。后台中棕色文字代表“仅管理员可见”的内部信息，对外不可见。

任务导航右侧“进展”面板已按最新原型整理：先看过程记录，再看时间记录、整体进展和交付验收，状态维护与最近操作记录作为低权重后台信息保留。

补录标记是公开解释信息，甲方需要看见；它使用绿色公开提示样式，不归入棕色管理员专属信息。

任务导航右侧“进展记录”现在是完整时间轴：进展更新、任务状态、排期调整和附件动态都在这里，管理员可见的具体时分仍用棕色标记。

整体进展滑块改为自定义绿色进度轨道和白底圆形滑块，拖动时更接近原型效果。

整体进展现在按 10% 档位吸附，并采用“未保存草稿 + 确认”机制：拖动或点档位不会立即写时间轴，只有点击确认才会保存。

管理员可以在任务导航右侧进展时间轴中删除单条任务动态，支持按钮和右键操作；删除仅清理时间轴记录，不回滚任务数据。

删除任务动态时会使用站内确认弹窗，不再出现浏览器原生确认框；确认文案会明确提示删除范围。

## 二、登录规则

打开 mayeai.com 会先看到登录页，有两种登录方式：

### 1. 管理员登录（你自己用）
- **邮箱**：`bh141425@gmail.com`
- **密码**：平台内部管理员密码，可在设置页修改
- 权限：最高权限，比普通用户多一个「口令管理」功能

⚠️ 注意：管理员必须同时输入邮箱和密码，两个都要正确才拥有最高权限。

### 2. 访问口令登录（其他设备 / 协作者用）
- **邮箱留空**，直接输入管理员发放的访问口令（`wk_` 开头）
- 权限：只读浏览为主，不能进行删除等危险操作；管理员才拥有完整管理权限

### 退出登录
左下角「设置」→「退出登录」。登录凭证存在浏览器里，**公共电脑用完务必退出**。

### 登录失效的几种情况
- 口令被管理员停用或删除 → 立即失效，回到登录页
- 口令到期 → 自动失效
- 重新登录即可（管理员账号永远有效）

### 想修改管理员密码怎么办
登录后进入 **设置 → 权限安全 → 账号安全**，填写当前密码和新密码即可。修改后不影响已发放的访问口令。

### 忘记密码怎么办
登录页点击「忘记密码？」后输入管理员邮箱。邮件找回需要先配置发信服务 `RESEND_API_KEY`；未配置时，系统会提示先通过已登录设备在设置页修改密码，或配置发信服务。

### AI 助手密钥
新建任务里的 AI 助手和工时建议使用 Cloudflare Worker Secret `DEEPSEEK_API_KEY` 调用 DeepSeek。密钥只保存在 Cloudflare 后端，前端页面不会暴露。

如果 AI 助手提示密钥未配置，需要给正式站写入 Secret：
- `npx wrangler secret put DEEPSEEK_API_KEY`

### 多模态模型兜底
设置页「AI 模型设置」现在支持四路配置：文字主模型、文字备用模型、识图主模型、识图备用模型。

默认策略：
- 文字任务：DeepSeek 优先，Kimi K2.6 兜底
- 识图任务：Gemini 3 Flash 优先，Kimi K2.6 兜底

每一路都可以配置供应商、Base URL、模型名称和 API Key，并可以直接在设置页点击「测试」确认是否可用。平台默认 Key 建议写入 Cloudflare Secret，例如 `GEMINI_API_KEY`、`KIMI_API_KEY`；多租户后也可以让租户在设置页自行填写并加密保存。

### BAML 与多模型路由
项目已引入 BAML 工程结构，AI 提示词和输出类型会先维护在 `baml_src/ai_assistants.baml` 中，并通过 `npm run baml:generate` 生成 TypeScript client。

当前正式站仍由 Cloudflare Worker 直接调用 DeepSeek。原因是当前 BAML TypeScript runtime 依赖 Node 原生模块，不能直接打包进 Cloudflare Worker；为了不影响正式站稳定性，BAML 暂作为 AI 契约和后续多租户模型路由的基础。后续多租户版本建议新增独立 AI Runtime / AI Gateway，再让用户按功能选择模型。

### 新建任务工时建议
新建任务时，在「预计开始时间 / 预估工时 / 预计交付时间」下方可以点击「AI 分析」获取工时建议。

系统会优先参考同类型历史任务的实际工时、交付周期、任务需求和验收备注；样本不足时会降低置信度，并提示已参考相近类型。建议结果会展示：
- 建议预估工时
- 置信度
- 历史样本数
- 平均工时和中位工时
- 主要分析依据

点击「采用建议」后，系统会把建议工时写入「预估工时」，并按当前排期锚点自动推算预计开始或预计交付时间。

## 三、口令管理（仅管理员可见）

位置：**设置 → 口令管理**

### 申请口令
1. 填备注（如"我的手机"、"iPad"、"协作设计师小王"）
2. 选有效期：**永久 / 7 天 / 30 天 / 90 天**
3. 点「申请口令」→ 自动复制到剪贴板，发给对方即可

### 管理口令
列表里每个口令显示：备注、口令本体（可再次复制）、创建时间、到期时间、最近使用时间、状态。

| 操作 | 效果 |
|------|------|
| 停用 | 该口令立即失效，用它登录的设备下一步操作就会被踢回登录页；可随时「启用」恢复 |
| 删除 | 彻底作废，不可恢复 |

| 状态 | 含义 |
|------|------|
| 有效 | 正常可用 |
| 已停用 | 被手动停用，可恢复 |
| 已过期 | 超过有效期自动失效 |

## 四、日常使用

### 顶栏（所有页面通用）
- **月份切换**：所有页面的数据都跟着所选结算月份走；月份选项根据任务的结算月份自动生成，永远包含当前月
- **结算**：从左侧导航进入结算页，核对月度结算单、锁定结算并生成甲方链接
- **新建任务**：随时可建
- **状态反馈**：操作成功、失败和处理中会用不同的提示样式；侧边栏标题下方的小圆点表示后端连接状态，异常时会变红。

### 工作台
当月总览：总工时、计费工时、预计收入、验收情况四张统计卡；任务明细列表（搜索框 + 筛选下拉，可按计划中/进行中/待验收/已验收过滤）；设计类型工时分布环形图、每周工时趋势图、年度统计柱状图都只统计实际工时，预计开始时间、预计交付时间和预估工时只作为排期参考。

### 任务
- **列表视图 / 日历视图**：进入任务页后，在页面内容区切换，不在左侧导航栏里展开。
- **列表视图**：任务列表会直接展示任务名称、需求摘要、对接人、实际工时、状态、进度、预计开始、预计交付和排期倒计时；搜索框在标题行右侧，状态筛选和「显示作废」在下一行，方便快速扫读。
- **排期提示**：任务行会根据预计开始和预计交付自动显示「距开始」「距交付」「今日 / 明日到期」「已逾期」或「已验收」。列表底部有「临期 / 逾期 / 进行中 / 正常」图例说明颜色含义。这些时间只用于排期提醒，不参与实际工时、收入统计、月报或最终结算。
- **信息 / 进展**：右侧详情区包含两个选项卡。信息页看基础字段；进展页按时间轴记录任务从接受、状态变化、时间段记录、文件上传到验收的全过程。
- **查看任务详情**：详情弹窗用于快速核对任务字段、进度和最近动态；右上角「待验收」状态悬停后会变为「去验收」，点击即可打开验收面板。底部只保留「进展」和「去编辑」两个可操作入口，关闭使用右上角图标。
- **弹窗关闭规则**：所有弹窗默认不会因为点击外部空白处或按 Esc 关闭，避免误点丢内容；需要关闭时点击右上角关闭图标或弹窗里的取消按钮。新建任务、记录进展和确认验收会保留未保存草稿，保存 / 确认后才清空。
- **右键快捷操作**：在工作台任务行或任务页列表中右键任务，可快速查看详情、记录进展、去验收、调整进度或复制已锁定月份的甲方分享链接；「状态」会以二级菜单收纳进行中、待验收、挂起和终止。挂起 / 终止会要求填写原因。
- **空白处右键新建**：在工作台任务列表或任务页列表空白处右键，可以直接打开「新建任务」菜单；未登录管理员时会先提示登录。
- **AI 优化进展 / 验收备注**：在「记录进展」弹窗的新增进展标题右侧、以及「确认验收」弹窗的验收备注标题右侧，都有 AI 图标。点击后，系统会结合当前填写内容、任务需求、任务相关文件、过程附件 / 验收附件和最近进展生成建议；建议需要手动点「采用建议」才会写回输入框。
- **进度与分段计时**：「记录进展」弹窗统一记录过程数据，包括整体进度、可结算分段计时、等待记录、进展说明和过程附件。分段计时会汇总为实际工时，用于后续验收和结算。
- **等待记录**：任务进展页的「时间记录」只记录可结算工时；「等待记录」用于记录等待甲方意见、补资料、等确认等非计费占用。等待记录不会进入结算工时，但会进入洞察，用于分析任务从接受到验收期间被项目占用但未产生收入的时间。
- **验收核对**：确认验收弹窗只负责终审核对和锁定结果。基础信息、整体进度、可结算分段计时、等待记录、验收附件与备注会分区展示；进度、分段计时和等待记录都来自「记录进展」，验收里不再临时添加或修改这些过程数据。
- **工时与等待汇总**：验收面板会按日期、时间段、内容和工时列出可结算分段计时，并显示实际总工时、后台时薪和本次预计结算金额；等待记录单独汇总，只用于洞察分析，不进入结算工时。
- **验收主观反馈**：确认验收时可选「顺利 / 一般 / 有问题」，并按需补充「需求不清晰 / 沟通成本高 / 定价偏低 / 技术挑战大」原因标签和体感评价。不填也能验收；这项只给管理员自己看，用于记录客观工时之外的任务体感。
- **删除任务动态**：在「记录进展」弹窗的进展时间轴中，管理员可右键单条动态，或悬停后点击删除图标，删除前会出现站内确认弹窗；删除只移除时间轴记录，不回滚任务字段、文件或工时数据。
- **任务不允许删除**：任务会影响工时、收入和税务统计，正式运营后不做删除。暂时不做的任务选择「挂起」，确定不做的任务选择「终止」并填写原因。
- **作废任务**：测试任务或误录任务可通过右键菜单作废，系统会弹出站内确认框，可填写作废原因；作废后不计入工时、收入和结算，也不出现在工作台、日历和月报里，但保留历史记录和关联文件。
- **查看 / 删除作废任务**：任务页默认隐藏作废任务，可打开「显示作废」查看；只有已作废任务才允许永久删除，删除前仍会二次确认。
- **排序规则**：工作台和任务列表先按“结算月份”筛选，再按“预计开始时间 / 开始日期倒序”排列；同一天内按创建时间倒序，新建的排在更上面。预计开始时间只影响列表排序和排期提醒，不参与统计、工时计算或结算。
- **计划时间**：新建任务时可填写预计开始时间到分钟，例如 `2026/06/13 10:10`。新建面板支持双向推算：打开「预计开始时间」旁的开关时，系统按开始时间 + 预估工时推算交付时间；打开「预计交付时间」旁的开关时，系统按交付时间 - 预估工时倒推开始时间。未打开开关的一侧只读展示。预计开始时间、预计交付时间和预估工时只用于排期参考，不影响本月洞察、实际工时、收入统计、月报或最终结算。
- **AI 优化需求**：新建任务时，「任务具体需求」右上角有 AI 图标。填写项目名称或原始需求后点击，可让 DeepSeek 助手把口语化描述整理成「设计背景 / 设计要求 / 输出文件」三段任务单语言，并推荐设计类型。建议只会展示给管理员，采用文案、采用分类或新增分类都需要手动确认。
- **补录历史任务**：新建任务弹窗右上角有「补录」文字和一个极简小开关，默认关闭。只有特殊补录场景才打开，打开后在开关旁显示「计入月份」下拉，不会占用主表单空间。例如任务实际 5 月完成，但需要放入 6 月工资一起结算，就把预计开始时间按真实 5 月填写，同时把计入月份选为 6 月。这样它会出现在 6 月工作台、任务列表和 6 月月度结算单中，并在日期旁标记“补录”。
- **结算月份**：普通新任务默认计入顶部当前月份；修改预计开始时间或预计交付时间不会自动改结算月份，需要调整归属时请显式修改「结算月份」。
- **补录状态建议**：新建任务不选择阶段，也不直接操作验收。已经完成、只需要补结算的任务，创建时填写必要的「补录说明」，创建后进入任务详情点击「确认验收」，再补实际工时、验收备注和验收文件。还没开始或正在做的任务，创建后按真实进度在任务详情或任务列表中调整状态。

### 任务状态规则

### 进展
- 进展已合并到任务详情里，不再作为左侧独立导航。
- 可以记录某个时间段做了什么，例如 `14:00-15:30 修改主视觉文案层级`。这些时间段会纳入任务实际工时。
- 状态变化、接受任务、上传文件、确认验收等操作也会写入时间轴。
- 确认验收时，如果已经修改验收备注、调整时间段或上传验收文件但还没点「确认验收」，关闭弹窗前会先提示确认；附件上传中或系统计算工时为 0 时不能提交验收。

### 文件库
- 文件库不是单独上传入口，而是自动汇总任务生命周期中的文件。
- 文件来源包括：任务过程文件、进展附件、验收时上传的最终稿和验收文件。
- 文件按“项目 / 任务”归档，点击某个任务后查看该任务下所有关联文件。
- 支持搜索文件名、任务名和标签；支持按月份筛选。
- 点击文件卡片会打开右侧信息栏，可重命名、添加多个标签、预览、打开或删除。
- 右键点击文件卡片可快速预览、打开、重命名、添加标签、下载源文件或删除；重命名和添加标签会自动聚焦右侧信息栏。
- 删除文件只用于处理误传文件。点击删除后会弹出站内二次确认框，确认后才会删除 D1 文件记录、R2 源文件和预览图。
- 已验收、已发给甲方或可能用于回溯的文件建议保留。
- 空格键可以快速预览当前选中文件。
- PSD、AI、PDF、Word、Excel、PPT、MP4 和图片尽量提供在线预览；部分源文件格式依赖浏览器和预览图能力，必要时仍可打开或下载源文件。

### 洞察（仅管理员可见）
- 工作台和任务管理列表会在部分任务行旁显示 **上下文洞察**，例如同类任务历史平均超时、当前预估低于同类均值、同类修改信号偏高等。它只在历史样本足够且信号明显时出现，悬停可查看依据。
- 左侧导航进入 **「洞察」**，顶部有「周期复盘 / 交付件理解 / 异常诊断 / 异常侦查」四个分区。
- 「交付件理解」展示附件的分析状态、模型摘要、需求匹配、质量问题、风险与建议。历史文件可点击「补分析历史附件」，失败文件可单独重试。
- 图片和 18MB 以内 PDF 可直接识别；35MB 以内 PPTX / DOCX / XLSX 当前基于文档文字和内嵌图片分析。PSD / AI 需要上传预览图，压缩包和旧版 Office 文件暂不自动解析。
- 「周期分析」可以按日、周、月、季度、半年和年度切换分析周期，查看周期任务数、验收率、实际工时、预估工时偏差、交付件数量、类型工时结构和进展投入趋势。
- 「交付件分析」会把任务、进展、上传文件和验收状态串起来，点击 PDF、JPG、PPT 等文件标签可以直接打开预览。
- 「能力画像」会展示设计类型覆盖、工时估算准确率、平均交付周期和对接人集中度，帮助判断效率稳定性和客户来源是否过度集中。
- 附件分析默认优先使用 Gemini 3 Flash，失败时切换 Kimi K2.6；文字助手仍优先使用 DeepSeek，失败时使用 Kimi K2.6。
- 后台每 5 分钟会继续处理 1 个未完成分析，单个失败任务最多自动尝试 3 次；升级 Cloudflare Paid plan 后可再提高单次处理量。
- 「异常侦查」：点击“运行 AI 诊断”后，系统会比较当前周期、上一对照周期、同类型历史基线和历史洞察，不再只罗列工时或占比。
- 每条结果都必须含“异常信号 / 数据证据 / 具体动作”；没有异常时会直说“本期无明显异常”。已说过但仍未解决的问题显示为“持续问题”，数据改善则显示为“已有改善”。
- 右侧「追踪中的洞察」会保留历史发现、建议、当时数据和状态。后台也会每 5 分钟用规则扫描异常，命中后再调用 AI 生成专项追踪建议。
- AI 洞察会读取验收主观反馈和原因标签，用来识别“工时正常但做得痛苦”的任务模式，例如某个对接人近期集中出现「沟通成本高」。
- AI 洞察会读取等待记录和任务周期，分析总周期小时、周期占用等待、明确等待小时和等待占比。等待占比高代表任务拖得久但计费少，可能挤占本可以接其他活的时间。
- 当前事件触发包括：同类任务修改信号偏高、本月工时比上月下降超过 20%、某个对接人综合时薪连续低于均值、某类设计类型超过 3 个月没有新任务、同类任务等待占比偏高。
- 修改轮次尚未单独录入时，系统采用进展文字中的“修改、调整、改稿、反馈、返工”等 **修改信号** 作为代理，不会把它伪装成精确轮次。

### 结算（月报核心流程）
月底结算的标准流程：

1. 切到要结算的月份，核对回执单上的任务和工时
2. 点 **「锁定结算并生成甲方链接」** → 金额快照存档，链接自动复制
3. 把链接（`mayeai.com/share/xxxx`）发给陈义君
4. 需要 PDF 的话点「导出 PDF」→ 打印窗口选「另存为 PDF」（文件名自动带月份）

**锁定规则：**
- 每个月只有一条结算记录；锁定后又改了数据，再点一次锁定即可更新金额，**链接不变**（已发出的链接不会失效）
- 锁定的金额是快照——以后改时薪不影响已锁定的月份
- 「结算历史」列出所有锁定过的月份，随时复制链接或打开查看
- **甲方查看回执**：甲方打开过链接后，结算历史会显示「甲方已查看 N 次（最近时间）」；没打开过显示「甲方尚未查看」——月底催验收前先看一眼

### 年度统计（工作台底部）
- 全年 12 个月的计费工时柱状图，🔒 表示该月已锁定结算
- 累计工时、累计收入一目了然（已锁定月份按结算快照计，未锁定月份按当前时薪实时算）
- 鼠标悬停柱子可看每月明细

### 甲方分享页（甲方看到的东西）
- 不需要任何口令，凭链接直接看
- 内容：锁定的工时与金额、任务明细（不含「不计费」任务）、该月份关联的进展和文件
- 文件点击弹窗在线预览；页面可直接打印成 PDF
- 甲方只能看到分享页内容，不能进入后台，也不能新增、修改或删除任何数据

### 结算回单 PDF
- 「结算」页下方的回单可通过「导出 PDF」打印保存。
- 回单包含结算摘要表和「工时明细附表」。
- 「工时明细附表」字段参考 Excel「6月」工时明细表结构，包括参考开始日期、设计类型、任务名称、需求、对接人、阶段、参考预估工时、实际工时、参考交付日期、实际交付日期、状态、验收人、风险/阻塞和进展；合计与金额只看实际工时。
- 明细内容来自平台任务、验收备注、时间记录和进展记录，不会直接复制 Excel 里的旧数据。

### 设置
- 左下角「设置」是管理员菜单，包含全站设置入口、R2 文件空间、退出登录和当前版本号。
- 点击左下角菜单里的「全站设置」进入完整设置页。

- **数据备份**：默认收纳在设置页的图标入口中，展开后可导出全部数据为 JSON 文件（建议每月锁定结算后顺手导一份）。
- **口令管理**：默认收纳在设置页的图标入口中，仅管理员可用。
- **账号安全 / 系统资源**：账号密码、Cloudflare 资源信息等低频内容仍在完整设置页中管理；退出登录和 R2 文件空间也可从左下角菜单快速查看。

## 五、计费规则速查

- 预计收入 = 计费工时 × 小时单价
- 计费工时 = 当月所有任务实际工时之和，**排除**「不计费」和「计划中」状态
- 统计只读取 D1 中的正式任务数据和已锁定月报快照，不再包含任何硬编码导入工时
- 结算单序号、人民币大写金额、电子印章均自动生成

## 六、运营与数据安全

### 正式站
- 正式站：`https://mayeai.com`，只录入真实任务、真实工时和真实交付文件。
- 预发布测试站已下线，测试 D1/R2 资源也已删除。
- 每次功能改动完成本地验证后直接同步正式站。

### 文件库清理原则
- 当前没有自动按月删除机制，避免误删当月文件或影响甲方月报。
- 推荐每月结算后执行一次人工复核：确认月报已锁定、甲方已验收、备份已导出，再考虑清理低价值过程文件。
- 最保险的做法不是直接删除，而是后续增加“归档 / 冷存储 / 待删除队列”机制，给文件至少 30 天反悔期。
- 最终稿、验收文件、已发给甲方的文件不建议删除；源文件是否删除应以“是否仍可能返修 / 补交”为准。
